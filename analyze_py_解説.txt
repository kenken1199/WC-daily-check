WC分析ツール　analyze.py コード解説
========================================
プログラム初心者向け：何がどこに書いてあるかをやさしく説明します


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 プログラムの全体構成 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  analyze.py は大きく7つのブロックに分かれています。

  1. ライブラリの読み込み（import）
  2. 定数の定義
  3. GUI（画面）の見た目の設定
  4. ロット分割プレビュー画面
  5. CSV読み込み・データ整形
  6. 統計計算
  7. エクセル出力
  8. ロット処理（グラフ作成・ファイル保存）
  9. ファイル処理・メイン画面

  これらは「関数（def）」や「クラス（class）」という単位で書かれています。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 ライブラリの読み込み　（1〜20行目あたり） 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  import ～ と書いてある行は「外部ライブラリの読み込み」です。
  Pythonには標準で多くの機能がありますが、足りない機能は
  他の人が作った「ライブラリ」を借りて使います。

  主なライブラリ：
  ・pandas      … 表形式データ（CSVなど）の読み込み・集計
  ・matplotlib  … グラフの描画
  ・numpy       … 数値計算（平均・標準偏差など）
  ・scipy       … 高度な統計計算（t値の算出など）
  ・tkinter     … Windowsの画面（GUI）を作るための標準機能
  ・openpyxl    … Excelファイルの読み書き


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 定数の定義　（20〜30行目あたり） 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  大文字で書かれた変数は「定数」です。プログラム全体で使いまわす
  固定値を一か所にまとめておくことで、後から変更しやすくなります。

  MIN_OK_COUNT = 2
    → 統計計算をするために必要なOK品の最小個数。
      これより少ないロットはスキップされます。

  _RANK_LABEL_MAP = {"2": "OK", "1": "軽量", "E": "過量", "0": "２個乗り"}
    → 重量チェッカーの内部コード（2, 1, E, 0）を
      人間が読める言葉（OK, 軽量, 過量, ２個乗り）に変換するための対応表。

  APP_BG, APP_TEXT, APP_ACCENT … など
    → 画面の色を決める定数。16進数カラーコード（#FAF6F0 など）で指定しています。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 _setup_app_style()　GUIの見た目を設定する関数 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ttk.Style という仕組みを使って、ボタン・ラベル・リストなど
  すべての画面部品の色・フォント・余白をまとめて設定しています。
  「clam」というテーマをベースに、アクセントカラー（青）などを上書きしています。

  ここを変えると → アプリ全体の見た目が変わります。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 LotPreviewDialog クラス　ロット分割プレビュー画面 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  「クラス」とは、画面や機能をひとまとめにしたものです。
  tk.Toplevel を継承（元にして作る）したポップアップ画面です。

  主なメソッド（クラスの中の関数）：

  __init__()
    → ダイアログが開いたときに最初に実行される。
      ボタン・リスト・ラベルなどの部品を画面に配置する。

  _compute_lots(threshold_min)
    → 「この分数以上データが空いたら別ロット」というルールで
      データを分割し、各行にロット番号を付ける。

  _update_preview()
    → しきい値ボタンを押したときに呼ばれる。
      _compute_lots を使って結果を再計算し、リスト表示を更新する。

  _on_ok()       → 「この分割でOK」ボタンが押された
  _on_no_split() → 「ロット分割しない」ボタンが押された
  _on_cancel()   → 「キャンセル」ボタンが押された


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 normalize_columns(file)　CSVを読み込んで整形する関数 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  重量チェッカーのメーカーによってCSVの形式が違うため、
  どのメーカーでも同じ列名・形式に統一するのがこの関数の役割です。

  対応メーカー：
  ・アンリツ … 「測定値(g)」「ランクコード」などの列名で見分ける
  ・イシダ   … 先頭10行を読み飛ばし、6列目のランク値で見分ける

  戻り値：
  ・整形済みのDataFrame（表形式データ）
  ・品種番号（CSVに記録されていれば）

  ポイント：
  どちらのメーカーでも最終的に
  「測定値出力No.・日付時刻・測定値(g)・ランクコード・メーカー」
  の5列に統一されます。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 analyze(data)　統計計算する関数 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  OK品の重さデータを受け取り、以下を計算して返します：
  ・平均（mean）
  ・標準偏差（std）
  ・信頼区間（ci）… 「真の平均がここにある可能性が高い範囲」
  ・最大値（max1）・最小値（min1）
  ・下限 −3σ（lower）・上限 +3σ（upper）

  2件未満のデータには計算できないため、その場合は None を返します。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 _style_3col_data_sheet()　エクセルシートのスタイル設定 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  「全OKデータ」「全外れ値（±3σ超）」シートで共通して使う
  スタイル設定（列幅・ヘッダー色・罫線・日付書式・フィルター）を
  ひとまとめにしたヘルパー関数です。
  同じコードを何度も書かなくて済むように切り出しています。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 _create_report_sheet()　「分析レポート」シートを作る関数 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Excelの「分析レポート」シートを1から組み立てる関数です。
  やっていることをざっくり順番に並べると：

  1. タイトル行（ネイビー色の帯）を書く
  2. 基本統計（全数・NG数・不良率など）を表形式で書く
  3. ランクコード集計表を書く
  4. ヒストグラム画像を貼り付ける（E列〜K列あたり）
  5. 全データ時系列グラフ画像を貼り付ける
  6. 時系列チャート（OK品のみ）画像を貼り付ける
  7. 印刷設定（A4縦・幅1ページ）を行う

  stats_rows というリストに「行ラベル・値・書式・警告フラグ」を
  まとめておき、ループで1行ずつExcelに書き込む構造になっています。
  "__subheader__" という特殊な値を入れると小見出し行として描画されます。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 save_to_excel()　Excelファイル全体を保存する関数 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Excelファイルに複数のシートを作り、スタイルを設定して保存します。
  pandas の ExcelWriter 経由でファイルを作り、そこに
  openpyxl で細かなデザインを追加しています。

  作成するシートの順番：
  1. 分析レポート   … _create_report_sheet() を呼んで作る
  2. 全データ時系列  … グラフ画像シート
  3. 全データ        … 全測定データ（ランクコード付き・NG行は赤）
  4. ヒストグラム    … グラフ画像シート
  5. 時系列チャート  … グラフ画像シート
  6. 全OKデータ      … OK品データ（外れ値行は赤）
  7. 全外れ値        … ±3σ超のデータ

  最後に sheet_order リストを使ってシートを正しい順番に並び替えています。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 process_lot()　1ロット分の分析をすべてやる関数 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  最もコード量が多い中心的な関数です。1ロット分のデータを受け取り、
  グラフ3枚を作成してExcelファイルに保存します。

  流れ：
  1. ランクコード集計（_RANK_LABEL_MAP を使って読める名前に変換）
  2. OK品データを抽出・数値変換
  3. OK品数が MIN_OK_COUNT 未満なら "skip" を返して終了
  4. analyze() で統計計算
  5. ±3σ外れ値の抽出
  6. 規格値（UCL/LCL）の計算（規格値が入力された場合のみ）
  7. 製造時間・開始・終了の計算
  8. ヒストグラム作成（matplotlib）→ PNG画像として保存
  9. 時系列チャート（OK品のみ）作成 → PNG保存
 10. 全データ時系列チャート作成 → PNG保存
 11. save_to_excel() を呼んでExcelファイルを保存
 12. "ok" を返す

  グラフは BytesIO（メモリ上の仮ファイル）に一時保存してから
  Excelに埋め込んでいます。実際のファイルには残りません。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 process_files()　ファイル選択からロット処理までをまとめる関数 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  「CSV選択して解析」ボタンを押したあとの処理全体を担います。

  流れ：
  1. ファイルパスを受け取り、フォルダならCSVを展開
  2. 各CSVを normalize_columns() で読み込み・統合
  3. 品種番号が違うCSVが混ざっていたら警告
  4. LotPreviewDialog を開いてロット分割を確認
  5. キャンセルなら終了
  6. ロットごとに process_lot() を呼ぶ
  7. 完了・スキップの件数を集計してメッセージ表示


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【 メイン画面（if __name__ == "__main__":） 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Pythonでは「if __name__ == "__main__":」以下が「このファイルを
  直接実行したとき」だけ動く部分です。

  やっていること：
  ・Tkウィンドウを作成
  ・タイトル「WC分析ツール」を設定
  ・「CSV選択して解析」ボタンを配置（押すと process_files が動く）
  ・画面を中央に配置
  ・コマンドライン引数でCSVが渡されたときは起動後に自動実行
  ・mainloop() でウィンドウを表示し続ける

【 データの流れ（全体まとめ） 】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  CSVファイル
      ↓ normalize_columns()  … メーカー差を吸収して統一形式に
  DataFrameに読み込む
      ↓ LotPreviewDialog    … ユーザーがロット分割を確認
  ロット別DataFrameに分割
      ↓ process_lot()
  ┌────────────────────────────────────────┐
  │  analyze()         … 統計計算          │
  │  matplotlib        … グラフ3枚を作成   │
  │  save_to_excel()   … Excelに書き出す   │
  └────────────────────────────────────────┘
      ↓
  デスクトップに .xlsx ファイルを保存
